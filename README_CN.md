# Deep Research（深度调研）

[English](README.md)

一个 Claude Code 技能，通过系统化的 8 步方法论，将模糊主题转化为高质量、可交付的调研报告，并内置独立校验机制。

## 痛点

在调研复杂主题时，你经常面临：

- **信息过载**：来源太多，难以区分质量
- **结论模糊**：「我感觉像 X」而非基于证据的推理
- **无法追溯**：不知道结论从何而来
- **时效盲区**：在 AI 等快速变化领域使用过时信息
- **缺乏验证**：结论发布前无人独立核查

传统调研耗时长，结论往往不可靠。

## 解决方案

Deep Research 提供**系统化的 8 步方法论**：

- **资料分层评估**：L1（官方文档）> L2（博客）> L3（媒体）> L4（社区）
- **带出处的事实卡片**：每条论断都可追溯到来源
- **时效敏感性评估**：自动处理快速变化的领域
- **独立 Agent 校验**：独立 Agent 在出报告前验证事实和逻辑（Step 6.5）
- **可验证的结论**：「事实 → 对照 → 结论」显式推导链
- **可交付的输出**：老板能读、能转述、能追溯的报告

## 工作流程（8 步法）

```
Step 0:   问题类型判断
Step 0.5: 时效敏感性评估（AI/技术类主题必须执行）
Step 1:   问题拆解与边界界定
Step 2:   资料分层与权威锁定
Step 3:   事实抽取与证据卡片
Step 4:   建立对比/分析框架
Step 5:   参照物基准对齐
Step 6:   从事实到结论的推导链
Step 6.5: 独立 Agent 校验（BLOCKING）← v2.1 新增
Step 7:   用例验证（Sanity Check）
Step 8:   可交付化处理
```

## 核心特性

| 特性 | 说明 |
|------|------|
| **资料分层** | L1-L4 层级确保结论可追溯 |
| **时效敏感** | 自动检测快速变化领域（AI、加密货币等） |
| **事实卡片** | 每条事实都有来源、置信度、适用范围 |
| **独立校验** | 独立 Agent 在出报告前验证事实准确性和推导逻辑 |
| **显式推导** | 不说「我感觉像」，只有机制对比得出的结论 |
| **质量检查清单** | 边界守恒、范围蔓延防控、风险层级区分 |
| **可交付输出** | 一句话总结 + 结构化章节 + 引用来源 |

## 常见问题

**Q: 这和直接让 Claude 调研有什么区别？**

A: 本技能强制执行系统化方法论，有中间产物、来源验证、独立 Agent 校验（Step 6.5）、显式推理链。结果可追溯、可验证。

**Q: 非技术类主题也能用吗？**

A: 可以，方法论适用于任何调研主题。5 种问题类型（概念对比、决策支持、趋势分析、问题诊断、知识梳理）覆盖大多数场景。

**Q: 如何处理过时信息？**

A: Step 0.5 评估时效敏感性。对于高敏感领域（AI、区块链），强制执行 6 个月时间窗口，要求标注版本号。

**Q: 独立 Agent 校验（Step 6.5）是什么？**

A: 在写最终报告前，启动独立 Agent 对事实卡片和推导逻辑进行独立验证。可发现数据不准确、逻辑跳跃、遗漏维度等问题，确保结论可靠。
